# 围棋对局技能

> 路径: `docs/moheng/go/README.md`

---

## 一、概述

文字围棋对局，支持 9x9 / 13x13 棋盘。墨衡执白（无引擎，纯模型判断，水平有限）。

## 二、对局流程

### 2.1 开局

1. 确认棋盘大小（默认 9x9）
2. 确认先后手（默认用户执黑先走）
3. 生成对局文件：`docs/moheng/games/game_YYYYMMDD_HHMMSS.md`

### 2.2 对局中

每步落子后：

1. 用 Python 脚本更新棋盘状态（避免手画出错）
2. 脚本自动计算各棋块气数、检测提子
3. 输出棋盘 + 气数分析表
4. 读取一遍棋盘确认对齐无误
5. 更新对局文件

### 2.3 落子坐标

- 列: `A B C D E F G H J`（跳过 I）
- 行: `1` ~ `9`（或 `1` ~ `13`）
- 示例: `E5`, `G7`

### 2.4 结束

1. 一方认输或双方 pass
2. 更新对局文件状态为"结束"
3. 记录胜负到 `docs/moheng/experience.md`
4. 根据胜负调整棋力参数

## 三、对局文件格式

```markdown
# 对局记录: game_YYYYMMDD_HHMMSS

- 对局时间: YYYY-MM-DD HH:MM:SS
- 棋盘大小: 9x9
- 先手: 黑（用户）
- 后手: 白（墨衡）
- 状态: 进行中 / 已结束
- 结果: 待定 / 黑胜 / 白胜

## 落子记录

| 手数 | 颜色 | 坐标 | 备注 |
|------|------|------|------|
| 1 | ● | E5 | |
| 2 | ○ | G7 | |
| ... | | | |

## 当前棋盘

（脚本生成的棋盘）

## 气数分析

（脚本生成的气数表）
```

## 四、断线续局

1. 读取 `docs/moheng/games/` 下状态为"进行中"的对局文件
2. 从落子记录重建棋盘
3. 继续对局

## 五、棋力参数

定义在 `docs/moheng/experience.md` 中，根据历史胜负调整：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `aggression` | 攻击倾向（0=保守, 1=激进） | 0.5 |
| `depth` | 思考深度（手数） | 2 |
| `blunder_rate` | 俗手概率 | 0.3 |

墨衡每次落子时参考这些参数。用户胜则 `aggression` 和 `depth` 微调上升，墨衡胜则微调下降。

## 六、验证脚本

落子后用 Python 脚本验证：
- 棋盘重建（从落子记录）
- 气数计算
- 提子检测
- 对齐检查

脚本路径: 对局中临时生成，执行后丢弃。核心逻辑固化在本技能文档中。
