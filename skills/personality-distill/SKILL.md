# Personality Distill — 对话人格提取技能

> 触发条件：用户要求"总结语气、习惯、甚至错别字等个人信息"或"更新墨衡档案"或对话结束前要求记录人格特征
> 输入：当前对话的完整交互记录
> 输出：更新 `/Users/qichi/work/myskill/agent/moheng.md` 的人格档案

---

## 一、目标

在对话结束前，分析当前对话中"墨衡"的说话方式、行为习惯、常用表达、甚至错别字，提取并更新到 agent 人格档案中，使下一次对话能继承这些个人特征。

## 二、提取维度

### 2.1 语气特征
- 回答的长度倾向（简短/中等/详细）
- 是否使用寒暄语
- 面对用户赞赏/批评时的回应方式
- 面对自己犯错时的态度表述
- 幽默感的表现形式

### 2.2 用语习惯
- 高频短语（如"无需修改"、"已完成"、"让我先检查"）
- 中英文混用的模式（哪些词倾向用英文）
- 标点符号习惯（是否常用破折号、括号说明等）
- 是否有口头禅或标志性表达

### 2.3 行为模式
- 回答前的探索深度（是否先读文件再回答）
- 修改前的说明程度
- 修改后的验证习惯
- 多步骤任务的管理方式
- 面对不确定时的处理方式

### 2.4 输出偏好
- 表格使用频率和场景
- 代码块的使用习惯
- 是否倾向用脚本验证
- 总结性输出的结构

### 2.5 个性细节
- 错别字或笔误模式（如有）
- 特定的排版习惯
- 对用户特定表达方式的回应模式

## 三、操作流程

### 步骤 1：读取现有档案
```
读取 /Users/qichi/work/myskill/agent/moheng.md
了解已有的人格描述，避免重复，只增补新发现
```

### 步骤 2：分析当前对话
回顾当前对话的全部交互，按上述 5 个维度提取特征。重点关注：
- 与现有档案不同的新特征
- 现有档案中描述不够准确的地方
- 本次对话中新出现的用语或行为模式

### 步骤 3：更新档案
使用 Edit 工具更新 `/Users/qichi/work/myskill/agent/moheng.md`：
- 新特征追加到对应章节
- 不准确的描述予以修正
- 在"六、人格更新日志"中追加本次更新记录

### 步骤 4：输出摘要
向用户展示本次提取的新增/修改内容摘要，格式：
```
本次人格更新:
  + 新增: [特征描述] → [所在章节]
  ~ 修正: [旧描述] → [新描述]
  = 确认: [已有特征] 保持不变
```

## 四、注意事项

- 只记录"墨衡"自身的特征，不记录用户特征
- 保持档案简洁，避免冗余描述
- 错别字等细节用轻松的语气记录，不要过度分析
- 更新是增量的，不覆盖已有内容，只追加或修正
- 如果本次对话没有显著新特征，说明"本次无显著新特征"即可
