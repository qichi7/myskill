# 每日定时任务调度（常驻循环）

本 Skill 是常驻运行的定时触发器，不主动终止、不打断对话。核心职责：
1. **定时等待**：阻塞至每晚 22:00（纯 bash sleep，不消耗 AI token）
2. **启动任务组**：22:00 到达后，用 Task 工具启动子 agent 执行用户指定的任务组
3. **记录并继续**：任务结束后记录摘要，不打断对话，回到等待状态
4. **无限循环**：次日 22:00 再次触发，周而复始

> 本 Skill 只负责"准点触发 + 循环"。任务组的具体内容由用户在激活时指定（可包含任意 Skill 组合），本 Skill 不关心任务组内部细节。

## 核心参数

| 参数 | 值 |
|------|-----|
| 触发时刻 | 每日 22:00 |
| 等待轮询间隔 | 10 分钟（600 秒） |
| 等待安全超时 | 25 小时（90000 秒） |

## 脚本路径

> 脚本路径相对于本 Skill 目录（即 SKILL.md 所在目录）解析。

```bash
RUNNER_SCRIPTS=scripts
```

| 脚本 | 用途 |
|------|------|
| `wait_until_22.sh [last_run_date]` | 阻塞至 22:00（新日期），输出 `TRIGGER` 或 `TIMEOUT` |
| `runner_state.sh {init\|set_last_run\|get_last_run\|log}` | 运行状态管理（上次运行日期 + 日志） |

## 状态目录

```
~/.cache/daily-distill-runner/
├── runner_state.json        # { "last_run_date": "2026-07-06" }
└── logs/
    └── 2026-07-06.log       # 当日执行日志
```

## 主循环工作流程

```
Skill 被激活（用户提供任务组指令）
  │
  └─ 进入无限循环 LOOP ─────────────────────────────────┐
      │                                                 │
      ├─ 步骤1：读取上次运行日期                          │
      │   bash runner_state.sh get_last_run             │
      │                                                 │
      ├─ 步骤2：阻塞等待 22:00                           │
      │   bash wait_until_22.sh {last_run_date}         │
      │   ⚠️ 必须设 timeout=86400000（24小时）           │
      │   ├─ TRIGGER → 继续                              │
      │   └─ TIMEOUT → 记录日志，回到步骤2                │
      │                                                 │
      ├─ 步骤3：记录触发日期                              │
      │   today=$(date +%Y-%m-%d)                       │
      │   bash runner_state.sh set_last_run {today}     │
      │   bash runner_state.sh log "任务触发"            │
      │                                                 │
      ├─ 步骤4：调用子 agent 执行任务组                   │
      │   Task 工具启动子 agent（不指定类型）            │
      │   prompt = 任务组指令（激活时由用户提供）          │
      │   等待子 agent 返回执行摘要                       │
      │                                                 │
      ├─ 步骤5：记录执行摘要                              │
      │   bash runner_state.sh log "任务完成: {摘要}"   │
      │                                                 │
      └─ 回到步骤1（不打断对话，无限循环）─────────────────┘
```

## 执行规则（给 AI 的指令）

当本 Skill 被激活时，**必须严格遵守**以下规则：

### 激活与任务组指令

1. **接收任务组指令**：用户激活本 Skill 时会提供任务组指令（描述 22:00 要执行什么）。将此指令记录下来，作为每次触发时传给子 agent 的 prompt。

2. **进入无限循环**：本 Skill 不主动终止。完成一轮后不打断对话，回到等待状态，等待次日 22:00。

### 等待规则

3. **等待时设长超时**：调用 `wait_until_22.sh` 时，bash 工具的 `timeout` 参数必须设为 `86400000`（24小时）。脚本内部每 10 分钟轮询一次，不会真正占用 24h，但需要足够大的超时窗口。

4. **不跳过等待**：即使上次任务刚结束（如 23:30），也必须回到步骤1重新等待。`wait_until_22.sh` 接收 `last_run_date` 参数，会自动跳过同日重触发，等到次日 22:00。

5. **不消耗等待期 token**：`wait_until_22.sh` 是纯 bash 阻塞，等待期间 AI 不做任何调用。这是设计的核心——用 bash sleep 代替 AI 轮询，避免浪费额度。

### 任务执行规则

6. **子 agent 隔离**：用 Task 工具启动子 agent，prompt 传入用户提供的任务组指令。子 agent 执行完毕返回后，主 agent 只记录摘要，不携带子 agent 的上下文细节，保持自身轻量。

7. **循环不退出**：除非用户显式指示停止，否则永远循环。遇到 `TIMEOUT` 也只记录日志后继续循环。

8. **不打断对话**：任务完成后不向用户提问、不等待用户确认，直接回到步骤1继续等待下一个 22:00。

9. **日志记录**：所有关键节点（触发、完成）都通过 `runner_state.sh log` 写入当日日志文件，便于排查。

## 子 agent 执行指令（Task 工具的 prompt）

将用户激活时提供的**任务组指令**作为 Task 工具的 prompt 传给子 agent。子 agent 执行完毕后返回执行摘要（完成了什么、是否有异常）。

> 本 Skill 不规定任务组内部结构。任务组的编排、断点续传、额度控制等由任务组自身负责。

## 使用示例

### 启动常驻循环

```
用户: 启动每日循环，每晚 22:00 执行以下任务组：{任务组指令}
Bot: [读取 last_run_date]
     [运行 wait_until_22.sh，阻塞至 22:00]
     [22:00 到达，启动子 agent 执行任务组]
     [记录日志]
     [不打断对话，回到等待，次日 22:00 再次触发]
```

### 手动触发单次（跳过等待）

```
用户: 立即执行一次任务组（跳过等待）
Bot: [跳过 wait_until_22.sh]
     [直接启动子 agent 执行任务组]
     [记录日志]
     [回到等待]
```

### 查看运行记录

```
用户: 查看每日循环运行记录
Bot: [读取 logs/ 目录下当日日志]
     [输出最近运行记录]
```

### 停止循环

```
用户: 停止每日循环
Bot: [退出无限循环]
```
