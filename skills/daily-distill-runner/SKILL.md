# 每日定时任务调度（常驻循环）

本 Skill 是常驻运行的定时触发器，不主动终止、不打断对话。核心职责：
1. **定时等待**：阻塞至每晚 22:00（纯 bash sleep，不消耗 AI token）
2. **执行指定 skill**：22:00 到达后，用 skill 工具加载并执行用户指定的 skill
3. **记录并继续**：任务结束后记录摘要，不打断对话，回到等待状态
4. **无限循环**：次日 22:00 再次触发，周而复始

> 本 Skill 只负责"准点触发 + 循环"。被调用 skill 的具体内容由用户在激活时指定，本 Skill 不关心被调用 skill 的内部细节。

## 核心参数

| 参数 | 值 |
|------|-----|
| 触发时刻 | 每日 22:00 |
| 等待轮询间隔 | 10 分钟（600 秒） |
| 等待安全超时 | 25 小时（90000 秒） |

## 脚本路径

> 脚本路径相对于本 Skill 目录（即 SKILL.md 所在目录）解析。

```bash
RUNNER_SCRIPTS=scripts
```

| 脚本 | 用途 |
|------|------|
| `wait_until_22.sh [last_run_date]` | 阻塞至 22:00（新日期），输出 `TRIGGER` 或 `TIMEOUT` |
| `runner_state.sh {init\|set_last_run\|get_last_run\|log}` | 运行状态管理（上次运行日期 + 日志） |

## 状态目录

```
~/.cache/daily-distill-runner/
├── runner_state.json        # { "last_run_date": "2026-07-06" }
└── logs/
    └── 2026-07-06.log       # 当日执行日志
```

## 主循环工作流程

```
Skill 被激活（用户提供主运行 skill）
  │
  ├─ 步骤0：立即触发检测
  │   若当前在 22:00-24:00 之间 → 跳过等待，直接进入步骤3
  │   否则 → 进入循环
  │
  └─ 进入无限循环 LOOP ─────────────────────────────────┐
      │                                                 │
      ├─ 步骤1：读取上次运行日期                          │
      │   bash runner_state.sh get_last_run             │
      │                                                 │
      ├─ 步骤2：阻塞等待 22:00                           │
      │   bash wait_until_22.sh {last_run_date}         │
      │   ⚠️ 必须设 timeout=86400000（24小时）           │
      │   ├─ TRIGGER → 继续                              │
      │   └─ TIMEOUT → 记录日志，回到步骤2                │
      │                                                 │
      ├─ 步骤3：记录触发日期                              │
      │   today=$(date +%Y-%m-%d)                       │
      │   bash runner_state.sh set_last_run {today}     │
      │   bash runner_state.sh log "任务触发"            │
      │                                                 │
      ├─ 步骤4：执行用户指定的 skill                      │
      │   用 skill 工具加载用户指定的 skill              │
      │   在主 agent 上下文中执行该 skill                │
      │   等待 skill 执行完成                            │
      │                                                 │
      ├─ 步骤5：记录执行摘要                              │
      │   bash runner_state.sh log "任务完成: {摘要}"   │
      │                                                 │
      └─ 回到步骤1（不打断对话，无限循环）─────────────────┘
```

## 执行规则（给 AI 的指令）

当本 Skill 被激活时，**必须严格遵守**以下规则：

### 激活与 skill 指定

1. **询问主运行 skill**：本 Skill 被激活后，第一件事是询问用户"每晚 22:00 要运行哪个 skill？"。用户回答后记录 skill 名称及其所需参数，作为每次触发时加载的目标。若用户在激活时已一并提供，则跳过询问。

2. **立即触发检测**：询问完主运行 skill、一切就绪准备开始等待前，立即检测当前时间：
   - 若当前在 22:00-24:00 之间（`$(date +%H)` ≥ 22），**跳过等待，直接执行步骤3**（记录触发日期 → 执行主 skill）
   - 否则进入正常等待循环

3. **进入无限循环**：本 Skill 不主动终止。完成一轮后不打断对话，回到等待状态，等待次日 22:00。

### 等待规则

4. **等待时设长超时**：调用 `wait_until_22.sh` 时，bash 工具的 `timeout` 参数必须设为 `86400000`（24小时）。脚本内部每 10 分钟轮询一次，不会真正占用 24h，但需要足够大的超时窗口。

5. **不跳过等待**：即使上次任务刚结束（如 23:30），也必须回到步骤1重新等待。`wait_until_22.sh` 接收 `last_run_date` 参数，会自动跳过同日重触发，等到次日 22:00。

6. **不消耗等待期 token**：`wait_until_22.sh` 是纯 bash 阻塞，等待期间 AI 不做任何调用。这是设计的核心——用 bash sleep 代替 AI 轮询，避免浪费额度。

### 任务执行规则

7. **直接执行，不通过子 agent 隔离**：用 skill 工具加载用户指定的 skill，在主 agent 上下文中直接执行。执行完成后记录摘要，不携带执行过程的上下文细节，保持自身轻量。

8. **循环不退出**：除非用户显式指示停止，否则永远循环。遇到 `TIMEOUT` 也只记录日志后继续循环。

9. **不打断对话**：任务完成后不向用户提问、不等待用户确认，直接回到步骤1继续等待下一个 22:00。

10. **日志记录**：所有关键节点（触发、完成）都通过 `runner_state.sh log` 写入当日日志文件，便于排查。

## 使用示例

### 启动常驻循环

```
用户: 启动每日循环，每晚 22:00 执行蒸馏skill（output_dir=~/distill/ops-transformer）
Bot: [询问主运行 skill → 蒸馏skill]
     [立即触发检测：若当前 22:00-24:00 → 直接执行]
     [否则运行 wait_until_22.sh，阻塞至 22:00]
     [22:00 到达，用 skill 工具加载蒸馏skill并执行]
     [记录日志]
     [不打断对话，回到等待，次日 22:00 再次触发]
```

### 手动触发单次（跳过等待）

```
用户: 立即执行一次蒸馏skill（跳过等待）
Bot: [跳过 wait_until_22.sh]
     [用 skill 工具加载蒸馏skill并执行]
     [记录日志]
     [回到等待]
```

### 查看运行记录

```
用户: 查看每日循环运行记录
Bot: [读取 logs/ 目录下当日日志]
     [输出最近运行记录]
```

### 停止循环

```
用户: 停止每日循环
Bot: [退出无限循环]
```
